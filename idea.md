# [기획안] 1인 가구 특화 AI 식단 및 식재료 관리 플랫폼 (Zero-Waste Kitchen)

## 1. 개요 및 페인 포인트 (Pain Point)
* **배경:** 1인 가구는 대량 구매한 식재료를 기한 내 다 쓰지 못하고 버리는 '식재료 폐기(음식물 쓰레기)' 문제와 "매일 뭐 먹지?"라는 식단 계획의 피로감을 동시에 겪음.
* **핵심 가치:** 공동구매의 오프라인 물류·대면 부담을 없애고, **[식단 계획 ➡️ 최적화 장보기 ➡️ 냉장고 잔여 재료 추적 ➡️ 꼬리물기 레시피]**로 이어지는 선순환 구조를 순수 소프트웨어(앱) 기술로 해결.
* **성공의 전제 (가장 중요):** 이 앱의 생명선은 **"가상 냉장고가 현실과 어긋나지 않는 것"**이다.
  냉장고 데이터가 실제와 달라지는 순간 유저는 앱을 신뢰하지 않고 이탈한다.
  따라서 모든 기능은 ① 입력을 자동화하고 ② 차감을 자연스럽게 만들고 ③ 매일 열어볼 이유를 주는 방향으로 설계한다.

---

## 2. 핵심 유저 여정 (User Journey)
1. **식단 설계:** 유저의 취향 및 목적(가성비 자취식, 저탄고지, 다이어트 등)에 맞춰 AI가 일주일 식단 자동 생성.
2. **최소한의 장보기:** 일주일 식단에 필요한 식재료 총량을 계산하여 마트에서 사야 할 최소 단위의 장바구니 리스트 추출.
3. **가상 냉장고 동기화:** 구매한 재료가 앱 내 '비주얼 냉장고'에 자동 등록되고, 신선도(추정 유통기한)가 추적됨.
4. **요리 & 자동 차감:** 앱이 추천한 레시피로 요리 후 "해먹었어요" 원터치 → 사용한 재료가 냉장고에서 자동 차감됨.
5. **꼬리물기 레시피:** 신선도가 임박한 남은 재료들을 조합해 소비할 수 있는 '냉장고 파먹기' 레시피를 AI가 선제적으로 추천.

> 💡 **설계 관점:** 레시피 추천은 단순 콘텐츠가 아니라 **냉장고 데이터를 갱신하는 입력 장치**다.
> "해먹었어요" 버튼이 차감 문제(수동 입력 귀찮음)를 푸는 핵심 열쇠이므로, 레시피와 냉장고는 반드시 한 몸으로 설계한다.

## 3. 차별화된 5대 킬러 기능 (기존 서비스 극복)

### 킬러 기능 1: 영수증 및 이커머스 구매 내역 자동 연동 (입력 제로화)
* **개념:** 유저가 식재료를 수동으로 입력해야 하는 귀찮음을 완벽히 해결. (기존 냉장고 관리 앱들이 실패한 1순위 원인 = 입력 피로)
* **동작:** 오프라인 마트 영수증을 사진으로 찍거나 쿠팡(로켓프레시), 이마트몰, 마켓컬리의 주문 내역 텍스트를 복사해 붙여넣으면 AI가 품목을 인식해 냉장고에 자동 등록.
* **기술 노트:**
  * 영수증에는 유통기한이 없다. 얻을 수 있는 것은 **품명 + 구매일**뿐이므로, 유통기한은 **품목 카테고리별 평균 보관기간으로 추정**한다. (예: 애호박 ~7일, 두부 개봉 후 ~3일)
  * 마트 영수증 품명은 "P)애호박1入" 같은 축약 코드라서 규칙 기반 파싱보다 **LLM 파싱**이 적합한 문제다.
  * 앱 전체 UX는 "추정 기반 신선도" 위에서 돌아간다는 것을 전제로 설계한다. (정확한 D-day가 아니라 신호등 단계로 소통)

### 킬러 기능 2: 유통기한 기반 '소비 우선순위' 신호등 시스템
* **개념:** 냉장고 안에서 썩어 나가는 재료를 직관적으로 시각화하여 소비 유도.
* **동작:** 신선도(추정)에 따라 **빨강(당장 소비), 노랑(3일 이내), 초록(여유)** 단계를 부여하고, 빨간 불이 들어온 재료들을 한 번에 털어낼 수 있는 맞춤형 레시피를 홈 화면에 최우선 노출. ("빨간 애들 털어먹기" 버튼 → 파먹기 레시피 직행)

### 킬러 기능 3: 비주얼 냉장고 — 귀여운 아이콘으로 보는 내 냉장고 (리텐션 엔진)
* **개념:** 냉장고 속 재료를 텍스트 목록이 아닌 **귀여운 아이콘이 배치된 냉장고 단면 그림**으로 보여준다. 관리 '의무감'이 아니라 다마고치처럼 돌보는 '재미'로 매일 열어보게 만들어, 냉장고 데이터가 현실과 동기화된 상태를 유지시키는 리텐션 장치.
* **공간 구성:** 실제 냉장고처럼 **냉장실 선반 3~4칸 + 문짝 포켓 + 냉동실 서랍**으로 나누고, 재료 등록 시 카테고리별 지정석에 자동 배치. (유제품은 문짝, 고기는 냉동 서랍, 채소는 아래 칸 — 실제 배치와 비슷할수록 "내 냉장고"라는 감각이 생김)
* **양(量) 표현 (조합 사용):**
  1. 기본: **채움 게이지** (아이콘 아래 작은 바 또는 반투명 처리) + **개수 뱃지** (계란 ×6)
  2. 상징 품목 몇 개만: **아이콘 자체 변형** (두부 한 모 → 반 모) — 직관성 최고, 제작량이 많으므로 일부 품목 한정
* **신선도 표현:** 색 테두리(초록/노랑/빨강)를 기본으로, 빨강 단계에서는 시든 표정·흔들림 애니메이션으로 "얘 좀 구해줘" 감정을 유도 → 파먹기 레시피 클릭으로 이어지는 감정 동선.
* **아이콘 수급 전략 (1인 개발):** 전 품목 아이콘 제작은 함정. **자취생 장바구니 상위 ~60개 품목**(계란, 두부, 대파, 양파, 삼겹살, 우유 등)만 전용 아이콘으로 만들고, 나머지는 카테고리 공용 아이콘("잎채소", "소스류") + 이름 라벨로 커버. AI 이미지 생성으로 일관된 스타일의 세트를 뽑아 다듬는 방식이 1인 개발에 적합.
* **주의:** 품목이 20개를 넘으면 아이콘 뷰는 오히려 검색성이 떨어진다. **아이콘 뷰(감성·재미) ↔ 리스트 뷰(검색·정확한 관리) 토글**을 반드시 함께 제공.

### 킬러 기능 4: 주요 이커머스 연동 장보기 (Deep Linking)
* **개념:** 식단 계획부터 최종 구매까지의 과정을 끊김 없이 연결.
* **동작:** AI가 추천한 장보기 리스트의 각 품목을 터치하면 쿠팡 등의 상품 검색/상품 페이지로 바로 이동 (어필리에이트 트래킹 링크).
* **기술 노트 (현실 반영):**
  * 쿠팡·컬리·이마트몰은 외부 앱이 장바구니에 직접 담는 공개 API를 제공하지 않는다. 실현 가능한 형태는 **"품목별 원터치 이동"**이며, "터치 한 번에 장바구니 완성"은 불가능하므로 기대 수준을 여기에 맞춘다.
  * 초기에는 **쿠팡 파트너스 하나로 시작**하는 것이 현실적. (컬리 등은 제휴 프로그램 문이 좁음)
  * 어필리에이트 수익은 링크 경유 + 일정 시간 내 구매에만 잡힌다는 조건을 수익 추정에 반영할 것.

### 킬러 기능 5: 냉파 챌린지 & '짠테크' 리포트 (차감 동기부여)
* **개념:** 식재료를 버리지 않고 끝까지 먹는 행위를 게임처럼 만들어 보상. "해먹었어요" 차감은 앱(데이터 정합성)에게만 좋고 유저에겐 즉각 보상이 없다는 빈틈을 메워, **유저 스스로 냉장고 신뢰 루프를 돌리게 만드는 연료** 역할.
* **동작:** 유통기한 임박(빨강/노랑) 재료를 요리해 먹고 "해먹었어요"를 누르면 '냉파 성공!'으로 기록. 월간 리포트로 "이번 달 식재료 98% 소비, 42,500원 절약" 같은 시각적 요약과 배지를 지급.
* **바이럴 연계:** 월간 리포트를 인스타 스토리 공유용 카드(요리 사진 + 아낀 금액 스탬프)로 생성 → 별도 소셜 피드 없이 외부 SNS에 얹혀가는 1인 개발형 성장 전략.
* **주의사항:**
  * 아낀 금액 계산은 **영수증에서 파싱한 실제 구매가 기반으로만, 보수적으로.** 과장된 환산(예: 환경 효과 뻥튀기)은 앱 전체의 신뢰(추정 유통기한 포함)까지 훼손한다.
  * **포인트→현금성 전환은 미룬다.** 포인트 제도는 정산·회계·약관 부담이 생기므로 초기엔 배지+리포트만. 쿠폰 전환은 어필리에이트 수익이 실제 발생한 뒤에 검토.

## 4. 1인 기업 관점의 비즈니스 모델 (수익화)
1. **커머스 어필리에이트 (제휴 수익):** 유저가 앱 내 링크를 통해 쿠팡 등에서 식재료를 결제할 때 발생하는 구매 금액의 2~3% 수수료 수취 (재고 및 배송 리스크 제로).
2. **프리미엄 식단 구독료:** 저당·저염 식단, 헬스 전문 식단 등 고도화된 맞춤형 서비스 제공 후 월 2,900~4,900원의 정기 구독료 수취.
   * ⚠️ **표현 주의:** "당뇨·고혈압 케어" 같은 질환 치료·관리 뉘앙스는 의료 광고 규제와 충돌할 수 있다. **"저당 식단 추천", "저염 식단 추천"** 수준의 표현으로 유지할 것.
3. **밀키트 브랜드 B2B 광고:** "이번 주 추천 식단" 탭에 특정 브랜드의 밀키트 제품을 노출하고 광고비 및 판매 전환 수수료 획득.

## 5. 기존 경쟁사 대비 우위 요소
* **기존 요리/레시피 앱 (어니언, 만개의 레시피 등):** 냉장고 재료 수동 입력이 귀찮고 구매 단계와의 연동이 없음.
* **기존 식단 앱 (플래딧 등):** 영양학적 관리에 치우쳐 자취생의 실질적인 '식재료 남음 및 가성비' 문제를 해결하지 못함.
* **삼성 푸드 (구 Whisk):** 레시피-식단-장보기 연동이라는 같은 그림을 그리고 있으나, 냉장고 **가전** 연동이 중심. 한국 자취생의 '가성비 파먹기' 감성과는 거리가 있음.
* **본 서비스의 무기:**
  1. 식단의 **[계획(Plan) ➡️ 구매(Buy) ➡️ 재활용(Cook)]** 전 과정을 한 네트워크 안에서 자동화하여 유저 이탈률(Churn Rate)을 낮춤.
  2. **자취생 특화 + 영수증 기반 입력 제로화 + 비주얼 냉장고 감성**이라는 각도는 기존 어느 서비스도 잡지 못한 포지션.

## 6. 레시피 엔진 아키텍처 (비용 계층화)

> 원칙: **싼 경로부터 쓰고, 비싼 LLM은 롱테일에만.** 그리고 LLM의 역할은 '창작자'가 아니라 **'변형자'**로 제한한다.

### 계층 1: 표준 요리 — 공공 데이터 기반 자체 DB (비용 0원)
* **데이터 소스:** 식약처 '조리식품 레시피 DB'(공공데이터포털, 약 1,100여 개). 영양 정보·저감(저염) 조리법이 포함되어 있어 향후 프리미엄 식단 기능과 궁합이 좋음.
* **사용 방식:** 실시간 API 호출이 아니라 **전체를 한 번 내려받아 자체 DB(Supabase)에 이관.** 정적 자료라 매번 호출할 이유가 없고, 자체 DB에 있어야 재료 기반 검색이 가능하다.
* **일회성 전처리:** 공공 DB의 재료 필드는 "돼지고기(목살) 100g" 같은 자유 텍스트이므로, LLM 일괄 처리로 **재료명을 정규화**해 둔다 (1,100개 일괄 처리 = 몇 천 원 수준). 이걸 해둬야 "내 냉장고의 두부+애호박으로 만들 수 있는 레시피" 쿼리가 가능해진다.

### 계층 2: 냉장고 파먹기 — 검색 후 변형 (Retrieval → Adaptation)
* **왜 "DB에 없으면 LLM 창작"이 아닌가:** 파먹기 쿼리는 내 냉장고의 정확한 재료 조합이라 거의 항상 "DB에 없는 조합"이다. 원안대로면 사실상 모든 파먹기 요청이 LLM 창작으로 넘어가 비용 절감이 무효화되고, LLM 백지 창작은 이상한 조합을 자신 있게 내놓는 품질 리스크가 있다.
* **동작:**
  1. 내 냉장고 재료와 **겹치는 재료가 가장 많은 공공 DB 레시피를 검색** (재료 커버리지 순 랭킹 — SQL 로직, 비용 0원)
  2. LLM에게는 **찾은 레시피를 내 냉장고에 맞게 고치는 역할만** 부여 ("호박이 없으니 양파로 대체하고 1인분으로 줄여줘")
* **효과:** 검증된 레시피가 베이스라서 **품질 하한선이 보장**되고, LLM 토큰 사용량도 창작 대비 크게 줄어든다.
* **캐싱:** 생성된 변형 레시피는 재료 조합을 키로 캐싱해 다른 유저에게 재사용 → **유저가 늘수록 LLM 호출 비용이 오히려 줄어드는 구조.**

### 계층 3: 유튜브 연동 — 검색 딥링크 (API 불필요)
* **저작권:** 영상을 가져오지 않고 링크만 제공하므로 안전. (원안의 판단 유지)
* **MVP 구현:** YouTube Data API의 검색은 호출당 100유닛, 무료 쿼터 하루 10,000유닛 = **하루 검색 100회뿐**이라 유저가 조금만 늘어도 막힌다. MVP에서는 API 없이 **유튜브 검색 결과 페이지 딥링크**(`youtube.com/results?search_query=자취생+계란볶음밥`) 버튼으로 대체 — 비용 0원, 쿼터 없음.
* **고도화:** 특정 영상을 썸네일로 골라 보여주는 것은 유저가 늘어 API 쿼터를 유료 증설할 가치가 생겼을 때 도입.

## 7. MVP 로드맵 (검증 순서)

가장 위험한 가설은 **"유저가 가상 냉장고를 계속 신뢰하고 쓰는가"**이다. 따라서 기능을 다 만들지 않고 아래 순서로 검증한다.

### 1차 MVP: 냉장고 신뢰 루프 검증 (킬러 기능 1 + 2 + 3 최소형)
* 범위: 영수증 사진 → LLM 파싱 → 냉장고 등록 → 신호등 → 빨간 재료 털기 레시피 → "해먹었어요" 차감
* 레시피 엔진은 6번 아키텍처의 최소형: 공공 DB 이관 + 재료 커버리지 검색 + LLM 변형 (유튜브는 검색 딥링크만)
* 비주얼 냉장고는 최소형으로: 칸 배치 + 기본 아이콘 + 색 테두리까지만 (애니메이션·변형 아이콘은 이후)
* 냉파 챌린지는 최소형으로: '냉파 성공' 카운트 기록까지만 (월간 리포트·공유 카드는 2차)
* **통과 기준: 2주 리텐션.** 유저가 2주 후에도 냉장고를 열어보고 있는가?

### 2차: 수익화 검증
* 1차 통과 후 식단 자동 생성 + 쿠팡 파트너스 링크를 붙여 어필리에이트 전환율 검증.

### 3차: 고도화
* 프리미엄 식단 구독, 밀키트 B2B 광고, 아이콘 변형·애니메이션 등 감성 고도화.
